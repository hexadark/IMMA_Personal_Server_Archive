# IMMA 프론트엔드 입력 명세서

각 화면에서 수집해야 하는 입력 필드, DB 저장 위치, 입력 방식, 연동 API를 매핑한 문서.

---

## 전체 흐름

```
[가입]           이름 + 이메일 + 비밀번호 (3개만)    → onboarding: draft
                   ↓
[온보딩 1]       장비 등록 (카탈로그 선택)            → 공정 역량 자동 생성
                   ↓
[온보딩 2]       가공 가능 재질 선택
                   ↓
[온보딩 3]       사업자등록번호 + 주소 입력            → onboarding: submitted
                   ↓
[검증]           사업자등록번호 확인                  → onboarding: verified
                   ↓
[매칭 활성]      company_capability_summary MV에 포함
```

매칭에 잡히려면 `onboarding_status = 'verified'` 필수.
MV 조건: `WHERE status = 'active' AND onboarding_status = 'verified'`

## 1. 발주자 가입

**API**: `POST /signup` (body에 `"role": "buyer"`)
**저장 테이블**: `imma.buyers`

가입 시 필수 2개만. 나머지는 프로필 편집에서 후속 입력.

| 화면 표시 | 필드명 | DB 컬럼 | 필수 | 입력 방식 | 입력 시점 |
|-----------|--------|---------|------|----------|----------|
| 이름/회사명 | `name` | `buyer_name` | **필수** | 텍스트 | 가입 |
| 이메일 | `email` | `email` | **필수** | 이메일 | 가입 (로그인 ID) |
| 비밀번호 | `password` | `password_hash` | **필수** | 비밀번호 | 가입 |
| 전화번호 | `phone` | `phone` | 선택 | 전화번호 | 프로필 |
| 담당자명 | `contact_name` | `contact_name` | 선택 | 텍스트 | 프로필 |
| 지역 | `region` | `region` | 선택 | 드롭다운 (시/도) | 프로필 |
| 회사 규모 | `company_scale` | `company_scale` | 선택 | 드롭다운 | 프로필 |

발주자는 온보딩 없이 바로 도면 발주 가능.

## 2. 업체(가공업체) 가입 + 온보딩

**가입 → 온보딩 → 검증**의 3단계. 온보딩 미완료 시 매칭에 안 잡힘.

### 2-1. 가입 (3초)

**API**: `POST /signup` (body에 `"role": "supplier"`)
**저장 테이블**: `imma.companies` + `imma.company_availability_snapshot` (자동)

| 화면 표시 | 필드명 | DB 컬럼 | 필수 | 입력 방식 |
|-----------|--------|---------|------|----------|
| 업체명 | `name` | `companies.company_name` | **필수** | 텍스트 |
| 이메일 | `email` | `companies.main_email` | **필수** | 이메일 (로그인 ID) |
| 비밀번호 | `password` | `companies.login_password_hash` | **필수** | 비밀번호 |

가입 시 자동 처리:
- `onboarding_status = 'draft'`
- `status = 'active'`
- `company_availability_snapshot` 자동 생성 (`overall_status = 'available'`)

---

### 2-2. 온보딩 1 — 장비 등록 (역량 자동 생성의 출발점)

가입 후 온보딩 첫 단계. **장비를 먼저 등록하면 공정 역량이 자동 생성된다.**

**저장 테이블**: `imma.equipment` + `imma.equipment_process_capabilities`
**자동 파생**: `imma.company_process_capabilities` (장비 카탈로그에서 자동 병합)
**연동 API**: `GET /api/equipment-categories` → `GET /api/equipment-models?category={code}`

| 화면 표시 | DB 컬럼 | 입력 방식 | 비고 |
|-----------|---------|----------|------|
| 장비 카테고리 | `equipment_category_code` | 드롭다운 (1단계) | `/api/equipment-categories` |
| 장비 모델 | `model_id` | 드롭다운 (2단계) | `/api/equipment-models?category=xxx` |
| 표시 이름 | `display_name` | 텍스트 | "CNC선반 1호기" 등 |
| 제조연도 | `year_made` | 숫자 | 선택 |
| 수량 | `quantity` | 숫자 | 기본값 1 |
| 현재 상태 | `status` | 드롭다운 | `running` / `idle` / `maintenance` / `down` |

**카탈로그 모델 선택 시 자동 채움** (equipment_model_catalog + equipment_catalog.json):
- 제조사, 모델명
- 최대 가공 크기 (X/Y/Z travel, 선삭 직경/길이)
- IT/Ra, 스핀들 RPM/kW
- **공정 역량 자동 생성**: 카탈로그의 `process_capabilities` → `company_process_capabilities`에 자동 INSERT
- **장비별 공정 능력 자동 생성**: → `equipment_process_capabilities`에 자동 INSERT

자동 채움 후 업체가 수정 가능 (실제 장비 상태 반영).
여러 장비를 등록할 수 있음 (1:N). 장비를 추가할 때마다 공정 역량이 누적 병합됨.

**예시 흐름**:
```
CNC선반 (SMEC SL 2000B) 선택
  → 자동 생성: turning IT7/Ra1.0, drilling IT9/Ra3.2, threading IT8/Ra1.6
  → 자동 생성: max_turning_diameter 298mm, max_turning_length 572mm

원통연삭기 (Jainnher JHU-2706CNC) 추가
  → 자동 추가: cylindrical_grinding IT5/Ra0.4
  → 자동 추가: grinding IT5/Ra0.4 (parent 공정)
```

### 2-3. 온보딩 2 — 가공 가능 재질

**저장 테이블**: `imma.company_material_capabilities`
**연동 API**: `GET /api/material-categories` → `GET /api/materials?category={code}`

| 화면 표시 | DB 컬럼 | 입력 방식 | 비고 |
|-----------|---------|----------|------|
| 재질 카테고리 | `material_category_code` | 드롭다운 (1단계) | `/api/material-categories`에서 목록 로드 |
| 구체 재질 | `material_id` | 체크박스 (2단계) | `/api/materials?category=xxx`에서 목록 로드 |
| 역량 수준 | `capability_level` | 드롭다운 | `regular` / `possible` / `consult` / `not_preferred` |

여러 재질을 등록할 수 있음 (1:N). 카테고리 단위 등록(`scope_type='material_category'`)도 가능하고, 구체 재질 단위(`scope_type='specific_material'`)도 가능.

재질은 장비에서 자동 유도가 안 되므로 수동 입력 필수.

---

### 2-4. 온보딩 2 — 추가 공정 등록 (장비 없이 수행하는 공정만)

장비 등록에서 자동 생성된 공정 외에, **외주 또는 장비 불필요 공정**을 수동 추가.

**저장 테이블**: `imma.company_process_capabilities`
**연동 API**: `GET /api/processes`

| 화면 표시 | DB 컬럼 | 입력 방식 | 비고 |
|-----------|---------|----------|------|
| 공정 | `process_code` | 체크박스 | 장비에서 이미 생성된 공정은 체크 표시 + 비활성 |
| 서비스 방식 | `service_mode` | 드롭다운 | `in_house` / `outsourced` / `both` |
| 평균 납기 (일) | `typical_lead_days` | 숫자 | 선택 |

**장비 없이 등록하는 대표 공정:**

| 공정 | 이유 |
|------|------|
| `casting` (주조) | 주물 공장에 외주 위탁 |
| `surface_treatment` (표면처리) | 도금/도장 전문 업체에 외주 |
| `heat_treatment` (열처리) | 자체 열처리로 없을 때 외주 |

이 공정들은 `service_mode='outsourced'`로 등록.

**기존 자동 생성된 공정의 오버라이드:**
장비에서 자동 채움된 IT/Ra/크기를 업체가 수정하고 싶으면 이 화면에서 편집 가능.

| 화면 표시 | DB 컬럼 | 비고 |
|-----------|---------|------|
| 최고 IT 등급 | `best_achievable_it_grade` | 자동 채움값을 수정 |
| 최고 Ra (μm) | `best_ra_um` | 자동 채움값을 수정 |
| 최대 크기 | `max_x/y/z_mm`, `max_turning_diameter/length_mm` | 자동 채움값을 수정 |

### 2-5. 온보딩 3 — 사업자 정보 + 주소 (verified 조건)

이 단계를 완료해야 `onboarding_status`가 `submitted` → `verified`로 전환되며 매칭에 잡힘.

**저장 테이블**: `imma.companies` + `imma.company_sites` + `imma.company_contacts`

#### 사업자 정보 (companies 업데이트)

| 화면 표시 | DB 컬럼 | 필수 | 입력 방식 | 비고 |
|-----------|---------|------|----------|------|
| 사업자등록번호 | `business_registration_no` | **필수** | 텍스트 | UNIQUE. verified 조건 |
| 대표자명 | `representative_name` | 선택 | 텍스트 | |
| 대표 전화 | `main_phone` | 선택 | 전화번호 | |
| 업체 규모 | `company_size` | 선택 | 드롭다운 | `micro`/`small`/`medium`/`large` |
| 종업원 수 | `employee_count` | 선택 | 숫자 | |
| 설립연도 | `established_year` | 선택 | 숫자 | 1900~2100 |
| 홈페이지 | `website_url` | 선택 | URL | |

#### 주소 (company_sites — 1건 자동 생성, `is_primary = true`)

| 화면 표시 | DB 컬럼 | 필수 | 입력 방식 |
|-----------|---------|------|----------|
| 시/도 | `region` | **필수** | 드롭다운 |
| 시/군/구 | `city` | 선택 | 텍스트 |
| 상세 주소 | `address_line1` | 선택 | 텍스트 |
| 우편번호 | `postal_code` | 선택 | 텍스트 |

#### 담당자 (company_contacts — 1건 자동 생성, `is_primary = true`)

| 화면 표시 | DB 컬럼 | 필수 | 입력 방식 |
|-----------|---------|------|----------|
| 담당자명 | `contact_name` | 선택 | 텍스트 | 미입력 시 기본값 '대표' |
| 직책 | `role_title` | 선택 | 텍스트 | |
| 전화번호 | `phone` | 선택 | 전화번호 | |
| 이메일 | `email` | 선택 | 이메일 | |

**온보딩 완료 조건** (onboarding_status → verified):
- 사업자등록번호 입력됨
- 장비 1대 이상 등록됨
- 재질 1종 이상 등록됨
- 시/도(region) 입력됨

조건 충족 시 자동으로 `onboarding_status = 'verified'` + MV REFRESH.
(`_check_onboarding` 함수가 매 온보딩 API 호출 시 자동 체크. rejected 상태는 우회 불가.)

### 온보딩 연동 API 경로

| 온보딩 단계 | API | 비고 |
|------------|-----|------|
| 2-2. 장비 등록 | `POST /api/equipment` | 카탈로그 연동 + 공정 자동 생성 |
| 2-3. 재질 등록 | `POST /api/material-capability` | materials/categories 배열 |
| 2-4. 추가 공정 | `POST /api/process-capability` | 외주 공정 수동 추가 |
| 2-5. 사업자정보 | `PUT /api/company/profile` | companies + sites + contacts UPSERT |
| 2-6. 가용상태 | `PUT /api/company/availability` | |

### 2-6. 가용 상태 (대시보드)

가입 시 기본값으로 자동 생성됨. 이후 업체가 대시보드에서 수시 변경.

**API**: `PUT /api/company/availability`
**저장 테이블**: `imma.company_availability_snapshot`

| 화면 표시 | DB 컬럼 | 입력 방식 | 비고 |
|-----------|---------|----------|------|
| 현재 상태 | `overall_status` | 드롭다운 | `available` / `limited` / `full` / `paused` |
| 다음 착수 가능일 | `next_available_date` | 날짜 | |
| 현재 가동률 (%) | `current_utilization_pct` | 숫자 | 선택, 0~100 |
| 최소 리드타임 (일) | `min_lead_time_days` | 숫자 | 선택 |

## 3. 도면 발주 (RFQ 생성)

2단계: **도면 업로드** → **발주 조건 입력**

### 3-1. 도면 업로드

**저장 테이블**: `imma.drawings`

| 화면 표시 | DB 컬럼 | 입력 방식 | 비고 |
|-----------|---------|----------|------|
| 도면 파일 | `file_uri` | 파일 업로드 | PDF/DXF/이미지 |
| 도면 번호 | `drawing_no` | 텍스트 | 선택 (VLM이 추출 가능) |

업로드 후 VLM이 자동 분석 → `vlm_result_jsonb`에 추출 결과 저장.

### 3-2. 발주 조건 입력

**저장 테이블**: `imma.rfqs` + `imma.rfq_parts` (VLM 분석 후 자동 생성)
**연동 API**: `GET /api/material-categories` → `GET /api/materials`, `GET /api/processes`

#### RFQ 헤더 (rfqs)

| 화면 표시 | 필드명 | DB 컬럼 | 입력 방식 | 필수 |
|-----------|--------|---------|----------|------|
| 희망 납기 | `due_date` | `requested_delivery_date` | 날짜 | 선택 |
| 수량 | `quantity` | `general_notes_jsonb` + `rfq_parts.quantity` | 숫자 | **필수** |
| 예산 | `budget` | `general_notes_jsonb` | 텍스트/숫자 | 선택 |
| 수령 방식 | `delivery_method` | `general_notes_jsonb` | 드롭다운 | 선택 |
| 용도 | `product_usage` | `general_notes_jsonb` | 텍스트 | 선택 |
| 추가 요청사항 | `additional` | `general_notes_jsonb` | 텍스트 (멀티라인) | 선택 |

`general_notes_jsonb` 저장 형식:
```json
{
  "client_notes": {
    "quantity": 10,
    "budget": "500만원 이내",
    "delivery_method": "택배",
    "product_usage": "산업용 감속기",
    "additional": "후처리 요청 등"
  }
}
```

#### 클라이언트 보정 입력 (VLM 추출 후 수정/보완)

VLM이 도면에서 추출한 결과를 보여주고, 클라이언트가 수정·보완할 수 있는 필드:

| 화면 표시 | DB 컬럼 | 입력 방식 | 비고 |
|-----------|---------|----------|------|
| 재질 | `rfq_parts.material_raw_text` | 드롭다운 2단계 또는 텍스트 | `/api/material-categories` → `/api/materials`. VLM 미추출 시 클라이언트 입력 필수 |
| 추가 공정 | `rfq_part_processes.process_code` | 체크박스 | `/api/processes`. VLM 추출 + 클라이언트 추가 |
| 후처리 | `rfq_parts.post_treatment_raw` | 텍스트 | 열처리/도금/도장 등 |

## 6. 데이터 흐름 요약

```
[발주자 가입] → buyers 1행
[업체 가입]   → companies 1행 + company_sites 1행 + company_contacts 1행
[역량 등록]   → company_material_capabilities N행
              + company_process_capabilities N행
              + equipment N행 + equipment_process_capabilities N행
              + company_availability_snapshot 1행
              → REFRESH MATERIALIZED VIEW company_capability_summary
[도면 발주]   → drawings 1행 → VLM 분석 → rfqs 1행 + rfq_parts N행 + rfq_part_processes N행
[매칭 실행]   → company_capability_summary 조회 → 후보 반환
[견적 회신]   → quote_responses 1행 + quote_line_items N행
[발주 확정]   → orders 1행
[리뷰 작성]   → reviews 1행
```

## 7. MV 갱신 시점

`company_capability_summary` Materialized View는 아래 시점에 REFRESH 필요:

| 시점 | 트리거 |
|------|--------|
| 업체 가입 완료 | 역량 등록 완료 후 |
| 업체 역량 수정 | 재질/공정/장비 변경 시 |
| 장비 상태 변경 | running → maintenance 등 |
| 리뷰 추가 | 평점 변동 |
| 가용 상태 변경 | available → full 등 |

현재는 수동 `REFRESH MATERIALIZED VIEW imma.company_capability_summary;`
구현 완료: 장비 등록, 재질 등록, 가용상태 변경, 리뷰 작성 시 자동 REFRESH 수행.
onboarding 상태 전환 시에도 verified 달성 시 자동 REFRESH.
